# Summary Report: “GenAI-powered App-DB Modernization workshop”

### Event Objectives

- Share best practices in modern application design, addressing the challenge of migrating from legacy architectures to the cloud.
- Introduce Domain-Driven Design (DDD), Event-Driven Architecture, and practical DevOps models.
- Provide guidance on selecting appropriate compute services (EC2, ECS, Fargate, Lambda) and modern tooling ecosystems.
- Shape core mindsets regarding the relationship between "Being a Human - Being a Professional - Being a Citizen" in the digital era, along with GenAI tools supporting the software development lifecycle (SDLC).

### Key Highlights

#### Identifying the drawbacks of legacy application architecture and the Role of Data Analytics

- Long product release cycles → Cause lost revenue, reduced GMV (Gross Merchandise Value), and missed market opportunities.  
- Inefficient operations → Reduce productivity, lack critical thinking when analyzing metrics to improve systems, and increase infrastructure costs.  
- Non-compliance with security regulations → High risk of severe data breaches, damaging corporate reputation on a global scale.  

#### Transitioning to modern application architecture – Microservices & DevOps

Migrating to a modularized system — where each function is an independent service communicating asynchronously via events, and operated with an automated DevOps mindset:
- Queue Management & CI/CD Pipelines: Automate continuous Build, Test, and Deploy processes.  
- Caching Strategy & Containerization: Apply Docker/Kubernetes to optimize performance and isolate resources.  
- Message Handling & IaC (Infrastructure as Code): Define infrastructure as code with flexible configuration management.  

#### Domain-Driven Design (DDD) and Real-world Case Study

- 4-step core method: Identify domain events → Arrange timeline → Identify actors → Define bounded contexts.  
- Bookstore Case Study & Operations System: Illustrate how to decompose complex business processes of large-scale systems.  
- Context Mapping: 7 pattern models to connect bounded contexts, aligning technical data flows with corporate operational streams.  

#### Event-Driven Architecture

- 3 key integration patterns: Publish/Subscribe, Point-to-point, Streaming (continuous data flows).  
- Practical benefits: Loose coupling, scalability, and automated resilience when failures occur.  
- Sync vs Async Comparison: Clear technical trade-offs; applying a No-Blame Post-Mortem culture during incidents (focusing on root-cause analysis rather than blaming engineers).  

#### Compute Evolution

- Shared Responsibility Model: Infrastructure progression from EC2 (Virtual Machines) → ECS/Fargate (Containers) → Lambda (Serverless).  
- Serverless & Optimization Philosophy: Remove server management overhead, auto-scale in real-time, and optimize costs based 100% on usage value.  
- Functions vs Containers: Select based on workload characteristics: use Functions for micro-architectures and fast execution; use Containers for complex systems and long-running states.  

#### Amazon Q Developer and GenAI in SDLC

- End-to-end automation: Support accelerating the software development lifecycle (SDLC) from planning and writing standardized code to optimizing queries and maintenance.  
- Code Transformation & Modernization: An effective assistant for upgrading Java, modernizing .NET, and supporting migrations of VMware and Mainframes to the cloud.  
- Smart AI usage: Maximize GenAI power to boost personal productivity while solidifying core fundamentals to avoid "shutting down the brain's critical thinking."  

### Key Takeaways

#### Design Mindset

- Business-first approach: Systems are born to solve business problems; technology is only a supporting tool.  
- Ubiquitous language: Bridge the vocabulary gap between Business and Tech teams. Build a Caring & Inclusive corporate culture environment.  
- Learn & Establish Standards: Lessons from Japan's Wakon Yosai model (Japanese Spirit, Western Technology) and South Korea's Chaebol model — mastering Western techniques, strictly adhering to the most stringent international standards while preserving core identity.  

#### Technical Architecture

- Event Storming Technique: A visual method connecting business experts and engineers to rapidly model entire processes.  
- Prioritize Asynchronous Communication: Reduce unnecessary direct synchronous calls to prevent domino effects that crash entire systems.  
- Master Fundamentals: Tool landscapes shift very quickly, but basic knowledge of Linux, Networking, Git, Python, Golang, and application structure remains a timeless core value.  

#### Modernization Strategy

- Roadmap-based approach: The infrastructure evolution of a nation from "isolated" to "ice-breaking innovation" and "Internet access" (1975 - 1997) is analogous to IT system upgrade roadmaps. Systems must be modernised in clear, distinct phases rather than all at once.  
- 7Rs Model & Digital Domino Chain: Capitalize on the digital infrastructure wave (Cloud, 4G, Smartphone) to establish Digital Factories.  
- Standard MNC Recruitment Processes: Apply a rigorous 4-step recruitment process (ATS screening -> Ability/Logic test -> Professional interview deep dive using the STAR model -> Core value compatibility assessment).  

### Applying to Work

- Apply DDD & Organize Event Storming: Plan to coordinate closely with business departments to map service boundaries (Bounded Contexts) for current projects.  
- Improve DevOps Model: Move beyond copying and pasting automation commands; proactively investigate root causes and ask "Why" before asking "How". Communicate effectively with development teams and avoid becoming a "midnight firefighting hero".  
- Implement Event-Driven & Serverless Architectures: Experiment with replacing synchronous APIs with asynchronous message queues, and pilot AWS Lambda for independent tasks.  
- Leverage GenAI Reasonably: Use Amazon Q Developer to accelerate coding and inspect bugs, but continuously study basic skills to maintain control over the technology.  
- Participate in Tech Communities: Engage deeply with the ecosystem through programs such as AWS Community Builder, First Cloud AI Journey, and AWS Student Builder Groups to continuously upgrade yourself and learn from experts.
